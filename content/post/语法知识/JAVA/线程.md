# 线程

# 以为的完美代码

## 一、处理耗时操作的方案

```java
public class SplashActivity extends AppCompatActivity {
    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_splash);
        
        // 将复杂计算移至子线程
        new Thread(() -> {
            performComplexCalculation();
            runOnUiThread(() -> {
                // 跳转到主界面
                startActivity(new Intent(this, MainActivity.class));
                finish();
            });
        }).start();
    }
}

```

## 二、客观评价

### ✅ **优点**：

1. **思路正确**：将耗时操作放到子线程
2. **简单直接**：代码简洁易懂

### ❌ **存在的问题和漏洞**：

#### **问题1：内存泄漏风险**

java

```
new Thread(() -> {
    performComplexCalculation();
    runOnUiThread(() -> {
        // 使用了"this" - 持有Activity引用
        startActivity(new Intent(this, MainActivity.class));
        finish();
    });
}).start();
```



- **问题**：匿名线程持有Activity的隐式引用（`this`）
- **风险**：如果Activity在计算完成前被销毁（如旋转屏幕），线程仍然持有引用，导致内存泄漏

#### **问题2：线程管理失控**

- **线程不会被自动销毁**：启动后会一直运行直到任务完成
- **无取消机制**：用户退出页面时，计算仍在继续
- **线程泄露**：任务完成后线程结束，但如果没有及时GC，相关资源可能泄露

#### **问题3：上下文理解错误**

java

```java
finish(); // 关闭的是什么？
```



- `finish()`关闭的是**SplashActivity本身**（UI界面）
- 调用位置在`runOnUiThread()`中，所以在**主线程**执行
- 与外部线程无关，外部线程执行完会自动结束

#### **问题4：没有错误处理**

- 网络异常、计算错误等情况没有处理
- 可能导致应用崩溃



### 1. **线程生命周期**

```java
new Thread(() -> { ... }).start();
```



- 线程启动 → 执行任务 → 任务完成 → 线程终止
- **但**：线程对象本身可能不会立即被回收
- **更好做法**：使用线程池管理

### 2. **线程嵌套限制**

```java
// 理论上可以无限创建线程，但实际有限制
new Thread(() -> {
    new Thread(() -> {
        new Thread(() -> { // 可以无限嵌套...
            // 但会导致：
            // 1. 资源耗尽
            // 2. 性能下降
            // 3. 难以管理
        }).start();
    }).start();
}).start();
```



**实际限制**：

- 每个线程占用~1MB内存（默认栈大小）
- 线程数过多会导致频繁上下文切换，性能下降
- 最佳实践：使用线程池控制并发数量

## 三、改进版本

```java
public class SplashActivity extends AppCompatActivity {
    private ExecutorService executor;
    private Future<?> futureTask;

    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_splash);
        
        // 使用单线程池
        executor = Executors.newSingleThreadExecutor();
        
        futureTask = executor.submit(() -> {
            try {
                performComplexCalculation();
                
                // 检查Activity是否还可用
                if (!isFinishing() && !isDestroyed()) {
                    runOnUiThread(() -> {
                        // 使用类名.this避免隐式引用问题
                        startActivity(new Intent(SplashActivity.this, MainActivity.class));
                        finish();
                    });
                }
            } catch (Exception e) {
                Log.e("SplashActivity", "计算失败", e);
                // 错误处理：跳转到错误页面或重试
            }
        });
    }

    @Override
    protected void onDestroy() {
        super.onDestroy();
        
        // 取消未完成的任务
        if (futureTask != null && !futureTask.isDone()) {
            futureTask.cancel(true);
        }
        
        // 关闭线程池
        if (executor != null) {
            executor.shutdownNow();
        }
    }
    
    private void performComplexCalculation() {
        // 模拟耗时操作
        try {
            Thread.sleep(3000);
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
        }
    }
}
```



## 四、实践建议

1. **使用线程池**而非直接创建Thread
2. **处理生命周期**：在onDestroy中清理资源
3. **使用弱引用**避免内存泄漏
4. **添加错误处理**
5. **考虑使用架构组件**：
   - ViewModel + LiveData
   - Kotlin协程 + LifecycleScope
   - WorkManager（适合后台持久任务）
6. **对于启动页的特殊考虑**：
   - 可以考虑不进行复杂计算，只做必要初始化
   - 可以使用Handler延迟跳转，模拟加载效果
   - 复杂的初始化放到MainActivity或Application中

# Thread.currentThread().interrupt() 的作用

### 1. **线程中断机制**

java

```
try {
    Thread.sleep(3000); // 可能被中断
} catch (InterruptedException e) {
    // 1. 恢复中断状态（重要！）
    Thread.currentThread().interrupt();
    
    // 2. 处理中断（如清理资源）
    cleanup();
    return;
}
```

### 2. **为什么要恢复中断状态？**

#### **正确的处理：**

java

```
catch (InterruptedException e) {
    // 恢复中断状态
    Thread.currentThread().interrupt();
    // 现在调用者可以检测到中断
}
```



### 3. **中断状态的工作流程**

java

```java
// 线程A（中断请求者）
threadB.interrupt(); // 设置中断标志

// 线程B（被中断者）
if (Thread.currentThread().isInterrupted()) {
    // 检测到中断，处理清理工作
    cleanup();
    return;
}

try {
    Thread.sleep(1000); // sleep()会检查中断标志
                        // 如果被中断，抛出InterruptedException
                        // 并清除中断标志
} catch (InterruptedException e) {
    // 中断标志被清除！需要重新设置
    Thread.currentThread().interrupt();
}
```



# Future 的作用和线程结合的使用

### 1. **Future 是什么？**

`Future` 代表**异步计算的结果**，它提供了：

- 检查计算是否完成
- 等待计算完成并获取结果
- 取消计算

### 2. **Future 的核心方法**

| 方法                                   | 作用             | 返回值             |
| :------------------------------------- | :--------------- | :----------------- |
| `boolean cancel(boolean mayInterrupt)` | 取消任务         | 是否成功取消       |
| `boolean isCancelled()`                | 任务是否被取消   | true/false         |
| `boolean isDone()`                     | 任务是否完成     | true/false         |
| `V get()`                              | 获取结果（阻塞） | 计算结果           |
| `V get(long timeout, TimeUnit unit)`   | 限时获取结果     | 计算结果或超时异常 |

### 3. **Future + 线程池的工作流程**

java

```Java
// 1. 创建线程池
ExecutorService executor = Executors.newSingleThreadExecutor();

// 2. 提交任务，返回Future
Future<String> future = executor.submit(() -> {
    Thread.sleep(2000);
    return "计算结果";
});

// 3. 可以继续做其他事情...
doOtherWork();

// 4. 需要结果时获取（会阻塞直到完成）
try {
    String result = future.get(); // 阻塞等待
    updateUI(result);
} catch (Exception e) {
    handleError(e);
}

// 5. 或者检查是否完成
if (future.isDone()) {
    // 获取结果
}
```



### **4.为什么需要 Future？**

**没有 Future 的问题：**

java

```
Thread thread = new Thread(() -> { ... });
thread.start();
// 无法：1. 取消任务 2. 检查状态 3. 获取结果
```



**有 Future 的好处：**

java

```java
Future<?> future = executor.submit(task);
// 可以：
future.cancel(true);    // 1. 取消
future.isDone();        // 2. 检查状态
future.get();           // 3. 获取结果（如果需要）
```



### 5. **Future 的实际使用场景**

#### **场景1：超时控制**

java

```java
try {
    // 最多等待3秒
    Object result = future.get(3, TimeUnit.SECONDS);
} catch (TimeoutException e) {
    // 超时处理
    future.cancel(true); // 取消任务
}
```



#### **场景2：批量任务管理**

java

```java
List<Future<String>> futures = new ArrayList<>();

// 提交多个任务
for (int i = 0; i < 10; i++) {
    Future<String> future = executor.submit(() -> {
        return processItem();
    });
    futures.add(future);
}

// 等待所有完成
for (Future<String> future : futures) {
    try {
        String result = future.get();
        // 处理结果
    } catch (Exception e) {
        // 处理异常
    }
}
```



#### **场景3：优雅取消**

java

```java
public class SplashActivity extends AppCompatActivity {
    private Future<?> splashTask;
    
    @Override
    protected void onCreate(Bundle savedInstanceState) {
        splashTask = executor.submit(() -> {
            while (!Thread.currentThread().isInterrupted()) {
                // 定期检查是否被中断
                performStep();
            }
            cleanup(); // 被中断时清理
        });
    }
    
    @Override
    protected void onDestroy() {
        if (splashTask != null) {
            // 发送中断信号
            splashTask.cancel(true);
        }
    }
}
```

### 总结

1. **主线程 = UI线程**：是同一个线程，负责所有UI更新
2. **Thread.interrupt()**：用于恢复中断状态，让调用者知道线程被中断过
3. **Future的作用**：
   - 控制异步任务的生命周期
   - 提供取消、状态检查、结果获取功能
   - 配合线程池实现优雅的资源管理
