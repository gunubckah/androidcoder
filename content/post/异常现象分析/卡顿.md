# 卡顿

## 一、卡顿原理

在Android中，卡顿通常是由于主线程（UI线程）被阻塞，导致无法在16ms内完成一帧的绘制（60fps）。原因可能包括：

- 

  主线程执行耗时操作（如网络请求、数据库读写、复杂计算等）

- 

  过度绘制导致GPU负载过高

- 

  内存不足导致频繁GC

- 

  视图布局过于复杂

- 

  动画使用不当



## **二、卡顿分析工具箱**

### **2.1 开发者选项工具**

| 工具                      | 作用               | 使用方式                       |
| ------------------------- | ------------------ | ------------------------------ |
| **GPU 渲染模式分析**      | 查看每帧渲染时间   | 开发者选项 → GPU 渲染模式分析  |
| **Profile GPU Rendering** | 图形化显示每帧耗时 | adb shell dumpsys gfxinfo      |
| **调试 GPU 过度绘制**     | 检查过度绘制情况   | 开发者选项 → 调试 GPU 过度绘制 |
| **严格模式**              | 检测主线程耗时操作 | 开发者选项 → 严格模式          |
| **显示布局边界**          | 查看视图层级       | 开发者选项 → 显示布局边界      |

### **2.2 命令行分析工具**

```bash
# 1. 查看应用帧率
adb shell dumpsys gfxinfo com.package.name framestats

# 2. 获取渲染性能数据
adb shell dumpsys gfxinfo com.package.name reset
# 操作应用...
adb shell dumpsys gfxinfo com.package.name

# 3. 查看主线程堆栈
adb shell am dumpheap com.package.name /data/local/tmp/heap.hprof

# 4. 监控CPU使用
adb shell top -d 1 | grep com.package.name

# 5. 内存监控
adb shell dumpsys meminfo com.package.name
```

## 三、分析方向

### **1.1 核心卡顿指标**

| 关键词                    | 含义         | 分析要点                    |
| ------------------------- | ------------ | --------------------------- |
| **Skipped frames**        | 跳过的帧数   | 超过60帧（1秒）表示严重卡顿 |
| **doFrame took**          | 单帧渲染耗时 | >16.67ms 表示掉帧           |
| **Choreographer**         | 渲染调度器   | 警告信息指示主线程繁忙      |
| **transaction too large** | 事务过大     | 通常>1MB，导致渲染延迟      |
| **Slow operation**        | 慢操作       | 系统检测到的耗时操作        |

### **1.2 主线程阻塞**

| 关键词                | 含义       | 分析要点              |
| --------------------- | ---------- | --------------------- |
| **main** (在traces中) | 主线程堆栈 | 查看阻塞位置          |
| **Binder**            | 跨进程通信 | 耗时>100ms 可能有问题 |
| **Lock contention**   | 锁竞争     | 线程等待锁释放        |
| **waiting on**        | 等待对象   | 查看等待的锁对象      |
| **blocked**           | 线程阻塞   | 线程状态为BLOCKED     |

### **1.3 内存相关卡顿**

| 关键词               | 含义       | 分析要点       |
| -------------------- | ---------- | -------------- |
| **GC_***             | 垃圾回收   | 频繁GC导致卡顿 |
| **Alloc concurrent** | 并发分配   | 内存分配压力   |
| **Pause**            | GC暂停时间 | >5ms 影响明显  |
| **lowmemory**        | 低内存     | 系统内存不足   |
| **kSwapd**           | 交换进程   | 内存交换频繁   |

### **1.4 系统资源限制**

| 关键词          | 含义     | 分析要点        |
| --------------- | -------- | --------------- |
| **thermal**     | 温度控制 | CPU降频导致卡顿 |
| **throttle**    | 频率限制 | CPU/GPU被限制   |
| **iowait**      | I/O等待  | 存储设备响应慢  |
| **irq latency** | 中断延迟 | 硬件响应延迟    |
| **scheduler**   | 调度器   | CPU调度问题     |
