# 代码混淆

## 一、混淆是什么？

**混淆** 是 Android 构建过程中的一个优化步骤，主要通过 ProGuard 或 R8 工具实现。它的**核心目标**是：

1. 

   **代码保护**：防止反编译后代码被轻易阅读

2. 

   **包体优化**：移除无用代码，减小 APK 体积

3. 

   **代码优化**：优化字节码，提升运行时性能





## 二、Android 中的混淆工具演进

| 阶段                       | 工具       | 特点                                    |
| -------------------------- | ---------- | --------------------------------------- |
| 早期                       | ProGuard   | 独立的 Java 字节码优化工具              |
| Android Gradle Plugin 3.4+ | R8（默认） | Google 开发的 ProGuard 替代品，速度更快 |
| 当前                       | R8         | 编译时进行脱糖、压缩、混淆、优化        |



## 三、启用混淆配置

在模块的 `build.gradle`中配置：

```groovy
android {
    buildTypes {
        release {
            // 1. 开启混淆
            minifyEnabled true
            // 2. 开启资源压缩（与混淆配合）
            shrinkResources true
            
            // 3. 指定混淆规则文件
            proguardFiles getDefaultProguardFile('proguard-android-optimize.txt'), 
                         'proguard-rules.pro'
            
            // 4. 开启代码优化（可选）
            proguardFiles getDefaultProguardFile('proguard-android.txt')
        }
        
        debug {
            minifyEnabled false
        }
    }
}
```

**文件说明**：

- 

  `proguard-android.txt`：Android 默认规则（基本规则）

- 

  `proguard-android-optimize.txt`：包含优化规则的版本

- 

  `proguard-rules.pro`：**自定义规则文件**



## 四、核心混淆规则详解

### 1. 保持类不被混淆（常用规则）

```pro
# 基本保持语法结构
-keep [,modifier,...] class_specification
```

**示例**：

```pro
# 保持特定类不被混淆
-keep public class com.example.MyClass

# 保持类及其所有成员不被混淆
-keep class com.example.MyClass { *; }

# 保持类名，但允许混淆成员
-keepnames class com.example.MyClass

# 保持类及其无参构造方法
-keep class com.example.MyClass {
    public <init>();
}
```

### 2. 保持类成员规则

```pro
# 只保持类成员，不保持类本身
-keepclassmembers class com.example.MyClass {
    <fields>;    # 保持所有字段
    <methods>;   # 保持所有方法
    <init>;      # 保持所有构造方法
}

# 保持特定成员
-keepclassmembers class com.example.MyClass {
    public static void main(java.lang.String[]);
    public void myPublicMethod();
    private void myPrivateMethod();
}
```

### 3. 条件保持规则

```pro
# 保持拥有指定成员的类
-keepclasseswithmembers class com.example.MyClass {
    public <init>(android.content.Context);
}

# 保持拥有指定成员的类名
-keepclasseswithmembernames class * {
    native <methods>;
}
```

## 五、按场景的混淆配置

### 1. 四大组件（必须保持）

```pro
# Activity
-keep public class * extends android.app.Activity
-keep public class * extends androidx.fragment.app.FragmentActivity

# Fragment
-keep public class * extends android.app.Fragment
-keep public class * extends androidx.fragment.app.Fragment

# Service
-keep public class * extends android.app.Service

# BroadcastReceiver
-keep public class * extends android.content.BroadcastReceiver

# ContentProvider
-keep public class * extends android.content.ContentProvider

# Application
-keep public class * extends android.app.Application

# 保持组件在 AndroidManifest 中的类名
-keepattributes *Annotation*
```

### 2. 序列化/反序列化

```pro
# 保持 Parcelable
-keep class * implements android.os.Parcelable {
    public static final android.os.Parcelable$Creator *;
}

# 保持 Serializable
-keepnames class * implements java.io.Serializable
-keepclassmembers class * implements java.io.Serializable {
    static final long serialVersionUID;
    private static final java.io.ObjectStreamField[] serialPersistentFields;
    private void writeObject(java.io.ObjectOutputStream);
    private void readObject(java.io.ObjectInputStream);
    java.lang.Object writeReplace();
    java.lang.Object readResolve();
}
```

### 3. 反射相关

```pro
# 反射调用的类
-keep class com.example.reflect.** { *; }

# 通过 Class.forName 调用的类
-keep class com.example.dynamic.** { *; }

# 注解处理器
-keepattributes RuntimeVisibleAnnotations, RuntimeVisibleParameterAnnotations
-keepattributes AnnotationDefault
```

### 4. 数据类/Bean/Model 类

```pro
# Gson
-keepattributes Signature
-keepattributes *Annotation*
-keepattributes EnclosingMethod

# 保持所有 Model 类（根据包名）
-keep class com.example.model.** { *; }

# 或者精确配置
-keep class com.example.model.User {
    <fields>;
    <methods>;
}
```

### 5. 网络框架

```pro
# Retrofit
-keepattributes Signature
-keepattributes *Annotation*
-keep class retrofit2.** { *; }
-dontwarn retrofit2.**
-keepclasseswithmembers class * {
    @retrofit2.http.* <methods>;
}

# OkHttp
-keepattributes Signature
-keepattributes *Annotation*
-keep class okhttp3.** { *; }
-dontwarn okhttp3.**
-keep interface okhttp3.** { *; }

# 保持回调接口
-keep interface com.example.api.** { *; }
```

### 6. 第三方库保持规则

```pro
# ButterKnife
-keep class butterknife.** { *; }
-dontwarn butterknife.internal.**
-keep class **$$ViewBinder { *; }
-keepclasseswithmembernames class * {
    @butterknife.* <fields>;
}
-keepclasseswithmembernames class * {
    @butterknife.* <methods>;
}

# EventBus
-keepattributes *Annotation*
-keepclassmembers class ** {
    @org.greenrobot.eventbus.Subscribe <methods>;
}
-keep enum org.greenrobot.eventbus.ThreadMode { *; }

# Glide
-keep public class * implements com.bumptech.glide.module.GlideModule
-keep public class * extends com.bumptech.glide.module.AppGlideModule
-keep public enum com.bumptech.glide.load.ImageHeaderParser$** {
  **[] $VALUES;
  public *;
}
```

### 7. 资源相关

```pro
# 保持 R 类资源 ID
-keepclassmembers class **.R$* {
    public static <fields>;
}

# 保持 Layout 文件中用到的 View
-keep public class * extends android.view.View
-keepclassmembers class * extends android.view.View {
    <init>(android.content.Context);
    <init>(android.content.Context, android.util.AttributeSet);
    <init>(android.content.Context, android.util.AttributeSet, int);
}

# 保持自定义属性
-keep class * extends android.util.AttributeSet
```

### 8. Native 方法

```pro
# 保持 JNI 调用的类
-keepclasseswithmembernames class * {
    native <methods>;
}

# 保持 JNI 接口
-keepclasseswithmembernames class * {
    public <methods>;
}
```

## 六、高级配置选项

### 1. 优化配置

```pro
# 启用优化
-optimizations !code/simplification/arithmetic,!field/*,!class/merging/*

# 优化时允许访问修改
-allowaccessmodification

# 合并接口
-mergeinterfacesaggressively
```

### 2. 混淆字典（提高混淆强度）

```pro
# 使用字典文件
-obfuscationdictionary dictionary.txt
-classobfuscationdictionary class_dictionary.txt
-packageobfuscationdictionary package_dictionary.txt

# dictionary.txt 示例
a
b
c
aa
ab
ac
# ... 可以自定义混淆后的名称
```

### 3. 保留行号（方便崩溃分析）

```pro
# 保留行号和源文件信息
-keepattributes SourceFile,LineNumberTable
-renamesourcefileattribute SourceFile
```

### 4. 移除日志

```pro
# 移除所有 Log 调用
-assumenosideeffects class android.util.Log {
    public static *** d(...);
    public static *** v(...);
    public static *** i(...);
    public static *** w(...);
    public static *** e(...);
}

# 移除自定义日志
-assumenosideeffects class com.example.MyLogger {
    public static *** d(...);
}
```

## 七、调试与验证

### 1. 查看混淆结果文件

构建完成后，在以下位置查看混淆相关文件：

```markdown
app/build/outputs/mapping/release/
├── mapping.txt      # 混淆映射关系（重要！用于反混淆）
├── seeds.txt        # 未被混淆的类/成员
├── usage.txt        # 被移除的代码
└── resources.txt    # 资源优化记录
```

### 2. 测试混淆后的应用

```bash
# 重新安装测试
./gradlew clean
./gradlew assembleRelease

# 或直接安装
adb install app/build/outputs/apk/release/app-release.apk
```

### 3. 反混淆崩溃堆栈

```bash
# 使用 retrace 工具
# 位于 Android SDK 的 proguard/bin/ 目录下

# 使用 mapping.txt 反混淆堆栈
retrace.sh -verbose mapping.txt stacktrace.txt
```

在 Android Studio 中反混淆：

```markdown
Tools → Android → ProGuard Unscramble
```

## 八、常见问题解决

### 1. 依赖库警告

```pro
# 忽略特定警告
-dontwarn okhttp3.**
-dontwarn javax.annotation.**
-dontwarn org.intellij.lang.annotations.**
-dontwarn org.jetbrains.annotations.**
```

### 2. 泛型类型擦除

```pro
# 保持泛型信息
-keepattributes Signature
-keepattributes InnerClasses
```

### 3. 枚举处理

```pro
# 保持枚举
-keepclassmembers enum * {
    public static **[] values();
    public static ** valueOf(java.lang.String);
}
```

### 4. 注解处理器

```pro
# 保持注解
-keepattributes *Annotation*
-keep @interface *
-keepclassmembers class * {
    @* *;
}
```

## 十、最佳实践建议

1. 

   **分模块配置**：大型项目为每个模块单独配置

2. 

   **测试验证**：混淆后必须进行全面测试

3. 

   **保留映射文件**：发布时备份 `mapping.txt`

4. 

   **渐进式混淆**：先保持所有，再逐步放开

5. 

   **监控崩溃**：集成崩溃监控（如 Firebase、Bugly），上传 mapping 文件

6. 

   **CI/CD 集成**：自动化混淆和 mapping 管理

```gradle
// 在 CI 中自动备份 mapping
android.applicationVariants.all { variant ->
    if (variant.buildType.name == "release") {
        variant.assembleProvider.get().doLast {
            copy {
                from variant.mappingFile
                into "$rootDir/mappings/"
                rename { fileName ->
                    "mapping-${variant.name}-${new Date().format('yyyyMMdd')}.txt"
                }
            }
        }
    }
}
```

混淆是发布前的重要步骤，合理配置能在保护代码的同时避免运行时问题。建议每次发版前都进行混淆测试。
