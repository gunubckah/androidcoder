# Android 依赖注入知识点总结

## 一、Android DI框架对比

| 框架       | 官方支持     | 编译时处理 | 运行时性能 | 学习曲线 | 适用场景             |
| ---------- | ------------ | ---------- | ---------- | -------- | -------------------- |
| **Hilt**   | ✅ Google官方 | ✅ 编译时   | ⭐⭐⭐⭐⭐      | 中等     | 推荐首选，现代化应用 |
| **Dagger** | ✅ Google维护 | ✅ 编译时   | ⭐⭐⭐⭐⭐      | 陡峭     | 大型复杂应用         |
| **Koin**   | ❌ 第三方     | ❌ 运行时   | ⭐⭐⭐        | 简单     | 小型项目，Kotlin优先 |
| **Kodein** | ❌ 第三方     | ❌ 运行时   | ⭐⭐⭐        | 简单     | Kotlin DSL风格       |

## 二、Hilt核心注解

| 注解                   | 作用范围       | 说明                             |
| ---------------------- | -------------- | -------------------------------- |
| **@HiltAndroidApp**    | Application    | 标记应用的Application类          |
| **@AndroidEntryPoint** | Android组件    | Activity/Fragment/Service/View等 |
| **@HiltViewModel**     | ViewModel      | ViewModel依赖注入                |
| **@Inject**            | 构造函数/字段  | 标记需要注入的依赖               |
| **@Module**            | 模块类         | 提供依赖的模块                   |
| **@InstallIn**         | 模块安装位置   | 指定模块安装在哪个组件           |
| **@Provides**          | Module方法     | 提供依赖实例的方法               |
| **@Binds**             | Module抽象方法 | 绑定接口到实现                   |

## 三、Hilt组件和作用域

| 组件                          | 作用域注解              | 生命周期           |
| ----------------------------- | ----------------------- | ------------------ |
| **SingletonComponent**        | @Singleton              | Application级别    |
| **ActivityRetainedComponent** | @ActivityRetainedScoped | Activity重建时保留 |
| **ActivityComponent**         | @ActivityScoped         | Activity生命周期   |
| **FragmentComponent**         | @FragmentScoped         | Fragment生命周期   |
| **ViewComponent**             | @ViewScoped             | View生命周期       |
| **ViewWithFragmentComponent** | @ViewScoped             | Fragment中的View   |
| **ServiceComponent**          | @ServiceScoped          | Service生命周期    |

## 四、依赖注入方式对比

| 注入方式         | 示例代码                                   | 优点             | 缺点                 |
| ---------------- | ------------------------------------------ | ---------------- | -------------------- |
| **构造函数注入** | `class Service @Inject constructor()`      | 依赖明确，不可变 | 只能用于自己创建的类 |
| **字段注入**     | `@Inject lateinit var service: Service`    | 简单直接         | 字段可变，非空检查   |
| **方法注入**     | `@Inject fun setService(service: Service)` | 注入后初始化     | 较少使用             |

## 五、常见组件注入示例

| 组件类型              | 注入示例                                                   |
| --------------------- | ---------------------------------------------------------- |
| **Activity**          | `@AndroidEntryPoint class MainActivity`                    |
| **Fragment**          | `@AndroidEntryPoint class MainFragment`                    |
| **ViewModel**         | `@HiltViewModel class MainViewModel @Inject constructor()` |
| **Service**           | `@AndroidEntryPoint class MyService`                       |
| **BroadcastReceiver** | 需通过@Assisted方式                                        |
| **ContentProvider**   | 不直接支持，需通过Application                              |
| **View**              | `@AndroidEntryPoint class CustomView`                      |

## 六、Hilt模块类型

| 模块类型             | 作用             | 示例                                            |
| -------------------- | ---------------- | ----------------------------------------------- |
| **@Module**          | 提供依赖的类     | `@Module @InstallIn(SingletonComponent::class)` |
| **@Binds** 模块      | 接口绑定         | 抽象类+@Binds方法                               |
| **@Provides** 模块   | 提供具体实例     | 类+@Provides方法                                |
| **@EntryPoint** 模块 | 非Hilt类获取依赖 | 用于非Hilt管理类                                |

## 七、依赖关系图

```
graph TD
    A[Application] -->|@HiltAndroidApp| B[ApplicationComponent]
    B -->|@Module| C[单例依赖]
    C -->|@InstallIn| B
    A -->|@AndroidEntryPoint| D[Activity]
    D -->|@AndroidEntryPoint| E[Fragment]
    D -->|@HiltViewModel| F[ViewModel]
    C -->|@Inject| D
    C -->|@Inject| E
    C -->|@Inject| F
```

## 八、Hilt + Dagger 工作流

1. 

   **定义依赖** - 用@Inject标记构造

2. 

   **提供依赖** - 在@Module中@Provides

3. 

   **安装模块** - 用@InstallIn指定作用域

4. 

   **标记入口** - 在组件用@AndroidEntryPoint

5. 

   **生成代码** - 编译时Dagger处理

6. 

   **自动注入** - 运行时Hilt注入

## 九、常见问题与解决

| 问题           | 原因           | 解决方案                |
| -------------- | -------------- | ----------------------- |
| **未找到绑定** | 未提供依赖     | 添加@Provides/@Binds    |
| **作用域冲突** | 作用域不匹配   | 检查@InstallIn作用域    |
| **编译错误**   | 依赖循环       | 使用@Assisted或Provider |
| **运行时Null** | 字段注入时机晚 | 使用by lazy或lateinit   |

## 十、最佳实践总结

| 实践                        | 说明                    |
| --------------------------- | ----------------------- |
| **构造函数注入优先**        | ViewModel、Repository等 |
| **字段注入用于Android组件** | Activity/Fragment       |
| **接口绑定用@Binds**        | 绑定抽象到实现          |
| **具体实例用@Provides**     | Retrofit、Room等        |
| **合理划分模块**            | 按功能/层级分组         |
| **避免全局作用域滥用**      | 只在必要时用Singleton   |

------

**一句话总结：Android DI首选Hilt，理解组件作用域是关键，构造函数注入优于字段注入，模块划分要清晰。**

## 十一、依赖注入生命周期对照表

| Android组件 | DI组件                    | 生命周期同步        |
| ----------- | ------------------------- | ------------------- |
| Application | SingletonComponent        | ✅ 同步              |
| Activity    | ActivityComponent         | ✅ 同步              |
| Fragment    | FragmentComponent         | ✅ 同步              |
| ViewModel   | ActivityRetainedComponent | ✅ ViewModel生命周期 |

## 十二、编译时vs运行时DI对比

| 对比维度     | 编译时DI(Hilt/Dagger) | 运行时DI(Koin/Kodein) |
| ------------ | --------------------- | --------------------- |
| **错误检测** | 编译时报错 ✅          | 运行时崩溃 ❌          |
| **性能**     | 运行时无开销 ✅        | 运行时查找开销 ❌      |
| **编译速度** | 注解处理耗时 ❌        | 编译快 ✅              |
| **灵活性**   | 代码生成复杂 ❌        | DSL简洁 ✅             |

------

**结论：大型项目选Hilt/Dagger保证稳定性，小型项目可选Koin简化开发。**
