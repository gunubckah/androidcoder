# 应用通信

在 **Android** 中，**通讯**主要可以**分为**以下几种类型： 用于组件之间的**通信**，特别是 Activity 和 Service 之间。 用于接收系统的广播消息。 用于在不同应用之间共享数据。 用于网络通讯，通过 TCP/UDP 协议进行数据传输。 在 **Android** 中，最常用的**通讯**方式之一是通过 Intent 进行的。

## 一、线程间通信

### 应用场景

1. 

   **后台任务更新UI**：网络请求、文件读写等耗时操作完成后更新界面

2. 

   **定时任务**：周期性的数据更新或界面刷新

3. 

   **任务协调**：多个线程间协作完成复杂任务

4. 

   **生产者-消费者模式**：数据生产和消费在不同线程

### 核心组件

#### 1. **Handler/Message/MessageQueue/Looper**

这是 Android 线程通信的**核心机制**。

```java
// 1. 在主线程创建 Handler
Handler mainHandler = new Handler(Looper.getMainLooper()) {
    @Override
    public void handleMessage(@NonNull Message msg) {
        switch (msg.what) {
            case MSG_UPDATE_UI:
                textView.setText((String) msg.obj);
                break;
        }
    }
};

// 2. 在子线程发送消息
new Thread(() -> {
    // 执行耗时操作
    String result = fetchDataFromNetwork();
    
    // 创建消息
    Message msg = Message.obtain();
    msg.what = MSG_UPDATE_UI;
    msg.obj = result;
    
    // 发送到主线程
    mainHandler.sendMessage(msg);
}).start();

// 3. 延迟消息
handler.postDelayed(() -> {
    // 3秒后执行
}, 3000);

// 4. 自定义 Looper 线程（HandlerThread）
HandlerThread handlerThread = new HandlerThread("WorkerThread");
handlerThread.start();
Handler workerHandler = new Handler(handlerThread.getLooper());
workerHandler.post(() -> {
    // 在 WorkerThread 执行
});
```

```java
// 过去常用的方式（API 30+ 已废弃）
private class DownloadTask extends AsyncTask<String, Integer, String> {
    @Override

    }
    
    @Override
    protected void onPostExecute(String result) {
        // 在主线程执行，任务完成后
        progressBar.setVisibility(View.GONE);

    }
}
```

#### 3. **Kotlin 协程（现代推荐方式）**

```kotlin
// 主线程启动协程
lifecycleScope.launch {
    // 在主线程执行
    showLoading()
    
    // 切换到 IO 线程执行耗时操作
    val result = withContext(Dispatchers.IO) {
        fetchDataFromNetwork()
    }
    
    // 自动切换回主线程

// 配合 Flow 进行流式通信
fun observeData(): Flow<String> = flow {
    for (i in 1..10) {
        delay(1000)
        emit("Data $i")
    }
}.flowOn(Dispatchers.Default) // 在后台线程发射

// 在 UI 线程收集
lifecycleScope.launch {
    observeData().collect { data ->
        // 在主线程接收数据

    }

```

#### 4. **LiveData（基于观察者模式）**

```kotlin
class UserViewModel : ViewModel() {
    // 这里，_userData是一个可变的LiveData，用于在ViewModel内部更新数据。而userData是一个对外暴露的不可变LiveData，这样外部只能观察而不能修改，保证了封装性。
    private val _userData = MutableLiveData<String>()
    val userData: LiveData<String> = _userData
    
    fun fetchUserData() {
        // viewModelScope是ViewModel的一个扩展属性，它是一个CoroutineScope，它会自动在ViewModel被清除时取消所有子协程，从而避免内存泄漏。
        viewModelScope.launch(Dispatchers.IO) {
            val data = repository.getUserData()
            _userData.postValue(data) // 切换到主线程  LiveData的setValue方法，则必须在主线程中调用。而postValue是线程安全的，可以在任何线程调用。
        }
    }
}

// Activity/Fragment 中观察
viewModel.userData.observe(viewLifecycleOwner) { data -> // viewLifecycleOwner是Fragment的生命周期所有者，这样可以确保观察只在Fragment处于活动状态时进行。当userData的值改变时，观察者会被调用，并且这个回调是在主线程中执行的，所以我们可以安全地更新UI。
    // 自动在主线程更新
    textView.text = data
}
```

## 二、进程间通信（IPC）

### 应用场景

1. 

   **应用间数据共享**：如通讯录、相册等数据共享

2. 

   **多进程应用**：WebView 独立进程、推送服务独立进程

3. 

   **系统服务调用**：如获取位置信息、发送通知

4. 

   **插件化/模块化架构**：各模块运行在不同进程

5. 

   **性能优化**：将内存密集型组件放到独立进程

### 核心组件

#### 2. **AIDL（Android Interface Definition Language）**

最强大的 IPC 方式，支持复杂数据类型的跨进程调用。

**步骤 1：定义 AIDL 接口**

```java
// IRemoteService.aidl
interface IRemoteService {
    int add(int a, int b);
    String processString(String input);
    void registerCallback(IRemoteCallback callback);
    void unregisterCallback(IRemoteCallback callback);
}

// IRemoteCallback.aidl
interface IRemoteCallback {
    void onDataChanged(in String data);
}
```

#### 3. **Messenger（基于消息的 IPC）**

适合**单向通信**，底层基于 Binder。

**服务端：**

```java
public class MessengerService extends Service {
    static final int MSG_SAY_HELLO = 1;
    
    class IncomingHandler extends Handler {
        @Override
        public void handleMessage(Message msg) {
            switch (msg.what) {
                case MSG_SAY_HELLO:
                    // 收到客户端消息
                    String clientMsg = msg.getData().getString("msg");
                    // 回复客户端
                    Messenger client = msg.replyTo;
                    Message reply = Message.obtain(null, MSG_REPLY);
                    Bundle data = new Bundle();
                    data.putString("reply", "Hello from Service!");
                    reply.setData(data);
                    try {
                        client.send(reply);
                    } catch (RemoteException e) {
                        e.printStackTrace();
                    }
                    break;
                default:
                    super.handleMessage(msg);
            }
        }
    }
    
    final Messenger messenger = new Messenger(new IncomingHandler());
    
    @Override
    public IBinder onBind(Intent intent) {
        return messenger.getBinder();
    }
}
```

**客户端：**

```java
public class ClientActivity extends Activity {
    private Messenger serviceMessenger;
    private Messenger clientMessenger = new Messenger(new ClientHandler());
    
    class ClientHandler extends Handler {
        @Override
        public void handleMessage(Message msg) {
            // 处理服务端回复
            String reply = msg.getData().getString("reply");
            textView.setText(reply);
        }
    }
    
    private ServiceConnection connection = new ServiceConnection() {
        @Override
        public void onServiceConnected(ComponentName name, IBinder service) {
            serviceMessenger = new Messenger(service);
            
            // 发送消息到服务端
            Message msg = Message.obtain(null, MSG_SAY_HELLO);
            Bundle data = new Bundle();
            data.putString("msg", "Hello from Client!");
            msg.setData(data);
            msg.replyTo = clientMessenger; // 设置回复的 Messenger
            
            try {
                serviceMessenger.send(msg);
            } catch (RemoteException e) {
                e.printStackTrace();
            }
        }
        
        @Override
        public void onServiceDisconnected(ComponentName name) {
            serviceMessenger = null;
        }
    };
}
```

#### 4. **ContentProvider**

**定义 ContentProvider：**

```java
public class UserProvider extends ContentProvider {
    private SQLiteDatabase db;
    
    @Override
    public boolean onCreate() {
        UserDbHelper helper = new UserDbHelper(getContext());
        db = helper.getWritableDatabase();
        return db != null;
    }
    
    @Override
    public Cursor query(Uri uri, String[] projection, String selection,
                       String[] selectionArgs, String sortOrder) {
        // 查询数据
        return db.query("users", projection, selection, 
                       selectionArgs, null, null, sortOrder);
    }
    
    @Override
    public Uri insert(Uri uri, ContentValues values) {
        long id = db.insert("users", null, values);
        return ContentUris.withAppendedId(uri, id);
    }
    
    @Override
    public int update(Uri uri, ContentValues values, String selection,
                     String[] selectionArgs) {
        return db.update("users", values, selection, selectionArgs);
    }
    
    @Override
    public int delete(Uri uri, String selection, String[] selectionArgs) {
        return db.delete("users", selection, selectionArgs);
    }
    
    @Override
    public String getType(Uri uri) {
        return "vnd.android.cursor.dir/vnd.com.example.users";
    }
}
```

## 四、网络通信

#### 1. **HttpURLConnection（原生）**

### 3. **NFC 通信**

```kotlin
// AndroidManifest.xml

// Activity 配置
<activity ...>

        <action android:name="android.nfc.action.NDEF_DISCOVERED" />
        <category android:name="android.intent.category.DEFAULT" />
        <data android:mimeType="text/plain" />

    </intent-filter>

        android:name="android.nfc.action.TECH_DISCOVERED"
        android:resource="@xml/nfc_tech_filter" />

// 处理 NFC 意图
override fun onNewIntent(intent: Intent) {
    super.onNewIntent(intent)
    
    if (NfcAdapter.ACTION_NDEF_DISCOVERED == intent.action) {
        val rawMessages = intent.getParcelableArrayExtra(NfcAdapter.EXTRA_NDEF_MESSAGES)
        rawMessages?.let {
            val messages = it.map { it as NdefMessage }
            // 处理 NDEF 消息
            val payload = messages[0].records[0].payload
            val text = String(payload, Charset.forName("UTF-8"))
            showNfcContent(text)
        }
    }
}

// 写入 NFC 标签
val nfcAdapter = NfcAdapter.getDefaultAdapter(this)
val writeIntent = Intent(this, javaClass).apply {
    addFlags(Intent.FLAG_ACTIVITY_SINGLE_TOP)
}

val pendingIntent = PendingIntent.getActivity(
    this, 0, writeIntent, PendingIntent.FLAG_MUTABLE
)

val writeTagFilters = arrayOf(IntentFilter(NfcAdapter.ACTION_TAG_DISCOVERED))

nfcAdapter.enableForegroundDispatch(
    this, pendingIntent, writeTagFilters, null
)

// 在 onNewIntent 中写入
val tag = intent.getParcelableExtra<Tag>(NfcAdapter.EXTRA_TAG)
val ndef = Ndef.get(tag)

ndef.connect()
val mimeRecord = NdefRecord.createMime(
    "text/plain", 
    "Hello NFC".toByteArray(Charset.forName("UTF-8"))
)
val ndefMessage = NdefMessage(arrayOf(mimeRecord))
ndef.writeNdefMessage(ndefMessage)
ndef.close()
```

## 六、通信方式选择指南

| 通信场景           | 推荐方案                    | 备注                            |
| ------------------ | --------------------------- | ------------------------------- |
| **线程间通信**     | Kotlin 协程 + Flow/LiveData | 现代 Android 开发首选，简洁安全 |
| **简单数据共享**   | ViewModel + LiveData        | 同一进程内组件通信              |
| **复杂跨进程**     | AIDL                        | 支持复杂数据类型和回调          |
| **简单跨进程消息** | Messenger                   | 基于消息队列，使用简单          |
| **跨应用数据共享** | ContentProvider             | 标准化的数据共享接口            |
| **应用内广播**     | LocalBroadcastManager       | 替代全局广播，更安全            |
| **网络请求**       | Retrofit + OkHttp           | RESTful API 标准方案            |
| **实时双向通信**   | WebSocket/Socket.IO         | 聊天、推送等场景                |
| **后台任务**       | WorkManager                 | 系统调度，保活能力强            |
| **蓝牙/NFC**       | Android 系统 API            | 硬件相关通信                    |
