# 接口设计

# 可扩展接口设计最佳实践

## 一、设计原则总结

| 原则         | 说明                   | 违反后果                 |
| ------------ | ---------------------- | ------------------------ |
| **开闭原则** | 对扩展开放，对修改关闭 | 每次扩展都要修改现有代码 |
| **里氏替换** | 子类可替换父类         | 继承体系混乱，无法多态   |
| **接口隔离** | 接口尽量小且专一       | 接口臃肿，实现类负担重   |
| **依赖倒置** | 依赖抽象而非实现       | 紧耦合，难以替换实现     |

## 二、接口设计模式对比

| 模式           | 适用场景     | 扩展性 | 复杂度 |
| -------------- | ------------ | ------ | ------ |
| **标记接口**   | 简单类型标记 | ⭐⭐     | 低     |
| **功能接口**   | 单一职责功能 | ⭐⭐⭐    | 中     |
| **构建器模式** | 复杂对象创建 | ⭐⭐⭐⭐⭐  | 高     |
| **策略模式**   | 算法族替换   | ⭐⭐⭐⭐   | 中     |
| **装饰器模式** | 动态添加功能 | ⭐⭐⭐⭐⭐  | 高     |

## 三、代码设计示例

### 1. **基础接口设计（错误示例）**

```java
// ❌ 错误设计：接口臃肿，扩展需修改接口
public interface UserService {
    User getUserById(Long id);
    User getUserByName(String name);
    User getUserByEmail(String email);
    User getUserByPhone(String phone);
    // 每新增一个查询条件就要修改接口
}
```

### 2. **优化方案1：参数对象模式**

```java
// ✅ 方案1：使用查询参数对象
public interface UserService {
    User getUser(UserQuery query);
}

// 查询参数对象
public class UserQuery {
    private Long id;
    private String name;
    private String email;
    private String phone;
    private Integer age;
    private String city;
    
    // 构建器模式
    public static class Builder {
        private UserQuery query = new UserQuery();
        
        public Builder id(Long id) {
            query.id = id;
            return this;
        }
        
        public Builder name(String name) {
            query.name = name;
            return this;
        }
        
        // 可以继续添加新属性，不影响现有代码
        public Builder age(Integer age) {
            query.age = age;
            return this;
        }
        
        public UserQuery build() {
            return query;
        }
    }
    
    // 使用方法
    User user = userService.getUser(
        new UserQuery.Builder()
            .id(1L)
            .name("张三")
            .age(25)  // 新增属性，不影响调用方
            .build()
    );
}
```

### 3. **优化方案2：泛型+配置对象**

```java
// ✅ 方案2：泛型接口 + 配置对象
public interface DataService<T, C extends QueryConfig> {
    List<T> query(C config);
    T getById(Object id);
    void save(T entity);
    void delete(Object id);
}

// 配置基类
public abstract class QueryConfig {
    private Map<String, Object> extensions = new HashMap<>();
    
    // 扩展属性支持
    public void setExtension(String key, Object value) {
        extensions.put(key, value);
    }
    
    public Object getExtension(String key) {
        return extensions.get(key);
    }
}

// 具体实现
public class UserQueryConfig extends QueryConfig {
    private String name;
    private String email;
    private Integer minAge;
    private Integer maxAge;
    
    // 原有属性getter/setter
    
    // 新增属性不影响基类
    private String department;
    
    public String getDepartment() {
        return department;
    }
    
    public void setDepartment(String department) {
        this.department = department;
    }
}
```
