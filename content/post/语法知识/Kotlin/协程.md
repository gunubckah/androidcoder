# 协程

### 1. **协程是什么？**

**协程是轻量级线程**，但不是线程。一个线程内可以运行成千上万个协程。

Kotlin 协程是一种轻量级的线程机制，用于优雅地处理异步任务。它通过挂起函数（*suspend*）和协程构建器（如 *launch* 和 *async*）实现非阻塞式的异步编程，简化了复杂的回调逻辑。

```kotlin
// 线程 vs 协程对比
fun compareThreadAndCoroutine() {
    // 1. 线程 - 重量级
    repeat(100_000) { i ->
        Thread { 
            Thread.sleep(1000L)
            println("Thread $i")
        }.start()  // 创建10万个线程 - 内存爆炸！
    }
    
    // 2. 协程 - 轻量级
    repeat(100_000) { i ->
        GlobalScope.launch {
            delay(1000L)  // 非阻塞挂起
            println("Coroutine $i")
        }  // 创建10万个协程 - 轻松应对
    }
}
```



### 2. **协程的核心概念**

#### **a. 挂起函数（Suspend Function）**

```kotlin
// 普通函数
fun doSomething() {
    // 阻塞当前线程
    Thread.sleep(1000)
}

// 挂起函数
suspend fun doSomethingSuspended() {
    // 挂起协程，不阻塞线程
    delay(1000)
    println("恢复执行")
}
```

#### **b. 协程构建器**

```kotlin
// 三种主要构建器
suspend fun main() {
    // 1. launch - 启动一个不返回结果的协程
    val job = GlobalScope.launch {
        println("launch: 后台任务")
    }
    
    // 2. async - 启动一个返回 Deferred 结果的协程
    val deferred = GlobalScope.async {
        println("async: 计算任务")
        "结果"
    }
    val result = deferred.await()  // 获取结果
    
    // 3. runBlocking - 阻塞当前线程，主要用于测试
    runBlocking {
        delay(1000)
        println("runBlocking: 阻塞测试")
    }
}
```

# 全面解析 viewModelScope 和 Kotlin 协程

## 一、viewModelScope 详解

### 1. **什么是 viewModelScope？**

`viewModelScope`是 **ViewModel 的 Kotlin 扩展属性**，它是 Android 架构组件的一部分，专门为 ViewModel 设计的协程作用域。

```kotlin
// 在 ViewModel 中
val ViewModel.viewModelScope: CoroutineScope
    get() {
        val scope: CoroutineScope? = this.getTag(JOB_KEY)
        if (scope != null) {
            return scope
        }
        return setTagIfAbsent(JOB_KEY, CloseableCoroutineScope(SupervisorJob() + Dispatchers.Main.immediate))
    }
```

### 2. **核心特性**

#### **a. 自动取消机制**

```kotlin
class MyViewModel : ViewModel() {
    fun fetchData() {
        viewModelScope.launch {
            // 当 ViewModel 被清除时，这个协程自动取消
            val data = repository.loadData()
            _liveData.value = data
        }
    }
    
    // ViewModel 的 onCleared 方法
    override fun onCleared() {
        super.onCleared()
        // 内部自动调用：viewModelScope.cancel()
    }
}
```

#### **b. 生命周期绑定**

```markdown
Activity/Fragment 生命周期：
onCreate → onStart → onResume → onPause → onStop → onDestroy
                                    ↓
                          ViewModel 的 onCleared
                                    ↓
                      viewModelScope 自动取消所有协程
```

### 3. **内部工作原理**

```kotlin
// 简化版的 viewModelScope 实现
class MyViewModel : ViewModel() {
    // 手动实现的类似 viewModelScope
    private val job = SupervisorJob()
    private val scope = CoroutineScope(Dispatchers.Main + job)
    
    override fun onCleared() {
        super.onCleared()
        scope.cancel()  // 取消所有协程
    }
    
    fun fetchData() {
        scope.launch {
            // 协程代码
        }
    }
}
```

## 三、viewModelScope 的实际应用

### 1. **基本用法**

```kotlin
class UserViewModel(
    private val userRepository: UserRepository
) : ViewModel() {
    
    private val _users = MutableLiveData<List<User>>()
    val users: LiveData<List<User>> = _users
    
    private val _loading = MutableLiveData(false)
    val loading: LiveData<Boolean> = _loading
    
    // 获取用户列表
    fun loadUsers() {
        viewModelScope.launch {
            _loading.value = true
            
            try {
                // 在IO线程执行网络请求
                val userList = withContext(Dispatchers.IO) {
                    userRepository.getUsers()
                }
                
                _users.value = userList
            } catch (e: Exception) {
                // 处理异常
                _users.value = emptyList()
            } finally {
                _loading.value = false
            }
        }
    }
    
    // 并行请求多个数据
    fun loadUserData(userId: String) {
        viewModelScope.launch {
            val userDeferred = async { userRepository.getUser(userId) }
            val postsDeferred = async { userRepository.getUserPosts(userId) }
            val friendsDeferred = async { userRepository.getUserFriends(userId) }
            
            // 等待所有并行请求完成
            val user = userDeferred.await()
            val posts = postsDeferred.await()
            val friends = friendsDeferred.await()
            
            // 合并数据
            _users.value = listOf(user.copy(posts = posts, friends = friends))
        }
    }
}
```

class SafeViewModel : ViewModel() {

                val data = repository.fetchData()

        }

        } catch (e: Exception) {
            Result.failure(e)
        }
    }

fun fetchUserData(continuation: Continuation): Any {

            return requestToken(continuation)
        }
        1 -> {  // 获取token后

            continuation.label = 2

        }
    }
}

## 六、实际项目架构示例

```kotlin
// 完整的 MVVM + 协程 + Flow 架构
class UserViewModel(
    private val userRepository: UserRepository,
    private val savedStateHandle: SavedStateHandle
) : ViewModel() {
    
    // StateFlow 状态管理
    private val _uiState = MutableStateFlow<UserUiState>(UserUiState.Loading)
    val uiState: StateFlow<UserUiState> = _uiState
    
    // 从保存的状态恢复
    private val userId: String = savedStateHandle["userId"] ?: ""
    
    init {
        loadUser()
    }
    
    // 使用 Flow 收集数据
    private fun loadUser() {
        viewModelScope.launch {
            // 从 Room 数据库观察数据变化
            userRepository.observeUser(userId)
                .flowOn(Dispatchers.IO)
                .catch { e ->
                    _uiState.value = UserUiState.Error(e.message ?: "未知错误")
                }
                .collect { user ->
                    _uiState.value = if (user != null) {
                        UserUiState.Success(user)
                    } else {
                        UserUiState.Error("用户不存在")
                    }
                }
        }
    }
    
    // 刷新数据
    fun refreshUser() {
        viewModelScope.launch {
            _uiState.value = UserUiState.Loading
            try {
                userRepository.refreshUser(userId)
            } catch (e: Exception) {
                _uiState.value = UserUiState.Error(e.message ?: "刷新失败")
            }
        }
    }
    
    // 密封类表示状态
    sealed class UserUiState {
        object Loading : UserUiState()
        data class Success(val user: User) : UserUiState()
        data class Error(val message: String) : UserUiState()
    }
}

// Repository 层
class UserRepository @Inject constructor(
    private val userApi: UserApi,
    private val userDao: UserDao
) {
    // 网络请求
    suspend fun refreshUser(userId: String) {
        val user = userApi.getUser(userId)  // 网络请求
        userDao.insertUser(user)  // 存入数据库
    }
    
    // 观察数据库变化
    fun observeUser(userId: String): Flow<User?> {
        return userDao.observeUserById(userId)
    }
}
```

## 总结

### **viewModelScope(协程作用域) 的核心价值：**

1. 

   **自动生命周期管理** - 与 ViewModel 生命周期绑定

2. 

   **避免内存泄漏** - ViewModel 销毁时自动取消协程

3. 

   **简化代码** - 不需要手动管理协程作用域

4. 

   **线程安全** - 默认在主线程启动

### **协程的关键特性：**

1. 

   **轻量级** - 一个线程可运行大量协程

2. 

   **结构化并发** - 父子协程关系明确

3. 

   **挂起非阻塞** - 提高线程利用率

4. 

   **流式API** - 支持 Flow 响应式编程

### **最佳实践原则：**

1. 

   始终在 ViewModel 中使用 `viewModelScope`启动协程

2. 

   耗时操作切换到 `Dispatchers.IO`

3. 

   UI 更新切换回 `Dispatchers.Main`

4. 

   正确处理异常和取消

5. 

   使用 Flow 进行响应式数据流处理

通过合理使用 `viewModelScope`和协程，可以构建出高效、可维护、无泄漏的 Android 应用架构。
