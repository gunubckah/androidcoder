# 广播

# Android 广播（Broadcast）全面总结

## 一、广播基本概念

| 概念             | 说明                                         |
| ---------------- | -------------------------------------------- |
| **广播**         | 系统或应用发送的全局消息，可以被多个组件接收 |
| **广播接收器**   | 接收广播的组件，继承 BroadcastReceiver       |
| **Intent**       | 携带广播信息的载体，包含 action、data 等     |
| **IntentFilter** | 过滤器，定义接收哪些广播                     |

## 二、广播类型对比

| 类型         | 特点                         | 应用场景           |
| ------------ | ---------------------------- | ------------------ |
| **标准广播** | 完全异步，所有接收器同时接收 | 普通事件通知       |
| **有序广播** | 同步执行，可按优先级传递     | 需要处理顺序的事件 |
| **本地广播** | 只在应用内传播，更安全高效   | 应用内部通信       |
| **粘性广播** | 发送后注册的接收器也能收到   | 已废弃，不推荐使用 |
| **系统广播** | 系统事件，如开机、充电等     | 系统状态监听       |

## 三、广播注册方式

### 1. **静态注册 vs 动态注册**

| 注册方式     | 声明位置            | 生命周期       | 适用范围 | 代码示例             |
| ------------ | ------------------- | -------------- | -------- | -------------------- |
| **静态注册** | AndroidManifest.xml | 应用安装到卸载 | 全局广播 | `<receiver>`标签     |
| **动态注册** | Java/Kotlin 代码    | 注册到取消注册 | 灵活控制 | `registerReceiver()` |

### 2. **静态注册示例**

```
<!-- AndroidManifest.xml -->
<receiver
    android:name=".MyBroadcastReceiver"
    android:enabled="true"
    android:exported="true">
    <intent-filter>
        <action android:name="com.example.MY_ACTION" />
        <action android:name="android.intent.action.BOOT_COMPLETED" />
    </intent-filter>
</receiver>
```

### 3. **动态注册示例**

```
// Activity/Fragment/Service 中
private val receiver = object : BroadcastReceiver() {
    override fun onReceive(context: Context, intent: Intent) {
        when (intent.action) {
            Intent.ACTION_BATTERY_LOW -> handleBatteryLow()
            "com.example.CUSTOM_ACTION" -> handleCustomAction(intent)
        }
    }
}

override fun onCreate() {
    super.onCreate()
    val filter = IntentFilter().apply {
        addAction(Intent.ACTION_BATTERY_LOW)
        addAction("com.example.CUSTOM_ACTION")
    }
    registerReceiver(receiver, filter)
}

override fun onDestroy() {
    super.onDestroy()
    unregisterReceiver(receiver)  // 必须取消注册
}
```

## 五、发送广播的方式

| 方法         | 说明               | 示例                                                         |
| ------------ | ------------------ | ------------------------------------------------------------ |
| **标准广播** | 所有接收器同时收到 | `sendBroadcast(intent)`                                      |
| **有序广播** | 按优先级顺序传递   | `sendOrderedBroadcast(intent, permission)`                   |
| **本地广播** | 应用内广播         | `LocalBroadcastManager.getInstance(context).sendBroadcast(intent)` |

### 发送示例：

```
// 发送标准广播
val intent = Intent("com.example.ACTION_REFRESH")
intent.putExtra("data", "Hello")
sendBroadcast(intent)

// 发送有序广播
val orderedIntent = Intent("com.example.ORDERED_ACTION")
sendOrderedBroadcast(orderedIntent, null)

// 发送带权限的广播
val secureIntent = Intent("com.example.SECURE_ACTION")
sendBroadcast(secureIntent, Manifest.permission.SECURE_BROADCAST)
```

## 六、有序广播处理

```
// 1. 发送有序广播
sendOrderedBroadcast(intent, null)

// 2. 接收器中可以
override fun onReceive(context: Context, intent: Intent) {
    // 获取数据
    val data = getResultData()
    
    // 修改数据
    setResultData("$data - processed")
    
    // 中止广播
    abortBroadcast()
    
    // 传递额外结果
    val extras = getResultExtras(true)
    extras.putString("key", "value")
}
```

## 七、本地广播（推荐用于应用内通信）

```
// 使用 LocalBroadcastManager
val localBroadcastManager = LocalBroadcastManager.getInstance(context)

// 发送本地广播
val intent = Intent("com.example.LOCAL_ACTION")
localBroadcastManager.sendBroadcast(intent)

// 注册本地广播接收器
localBroadcastManager.registerReceiver(receiver, IntentFilter("com.example.LOCAL_ACTION"))

// 取消注册
localBroadcastManager.unregisterReceiver(receiver)
```

## 八、权限控制

| 权限控制     | 发送端               | 接收端   |
| ------------ | -------------------- | -------- |
| **发送权限** | 指定接收器需要的权限 | 声明权限 |
| **接收权限** | 指定谁可以接收       | 声明权限 |

```
<!-- 1. 定义权限 -->
<permission android:name="com.example.MY_BROADCAST_PERMISSION" />

<!-- 2. 发送带权限的广播 -->
<receiver
    android:name=".MyReceiver"
    android:permission="com.example.MY_BROADCAST_PERMISSION">
    <intent-filter>
        <action android:name="com.example.PROTECTED_ACTION" />
    </intent-filter>
</receiver>

<!-- 3. 接收带权限的广播 -->
<uses-permission android:name="com.example.MY_BROADCAST_PERMISSION" />
```

## 九、最佳实践

### 1. **使用建议**

| 场景           | 推荐方式              | 原因               |
| -------------- | --------------------- | ------------------ |
| 应用内组件通信 | LocalBroadcastManager | 安全、高效         |
| 系统事件监听   | 动态注册              | 按需注册，节省资源 |
| 后台事件处理   | 静态注册              | 应用未启动也能接收 |
| 高频率事件     | EventBus/RxJava       | 避免广播性能开销   |
| 跨应用通信     | 带权限的广播          | 确保安全性         |

### 2. **代码模板**

```
// 广播接收器基类
abstract class BaseBroadcastReceiver : BroadcastReceiver() {
    
    override fun onReceive(context: Context, intent: Intent) {
        try {
            onReceiveBroadcast(context, intent)
        } catch (e: Exception) {
            Log.e("Broadcast", "处理广播异常", e)
        }
    }
    
    abstract fun onReceiveBroadcast(context: Context, intent: Intent)
}

// 使用
class MyReceiver : BaseBroadcastReceiver() {
    override fun onReceiveBroadcast(context: Context, intent: Intent) {
        // 具体处理
    }
}
```

## 十、Android 8.0+ 限制

| Android 版本 | 限制内容                       | 解决方案                                                 |
| ------------ | ------------------------------ | -------------------------------------------------------- |
| **8.0+**     | 大部分静态注册的隐式广播被禁止 | 1. 使用动态注册 2. 使用 JobScheduler 3. 使用 WorkManager |
| **例外广播** | 部分系统广播仍可静态注册       | 如：BOOT_COMPLETED, USER_PRESENT 等                      |

**豁免列表**（可静态注册）：

- 

  `ACTION_BOOT_COMPLETED`

- 

  `ACTION_LOCKED_BOOT_COMPLETED`

- 

  `ACTION_USER_INITIALIZE`

- 

  `ACTION_USER_PRESENT`

- 

  `ACTION_TIMEZONE_CHANGED`

- 

  `ACTION_TIME_SET`

## 十一、替代方案比较

| 通信方式           | 适用场景         | 优点               | 缺点                 |
| ------------------ | ---------------- | ------------------ | -------------------- |
| **广播**           | 全局、跨组件通信 | 系统支持，跨进程   | 性能一般，8.0+有限制 |
| **LocalBroadcast** | 应用内通信       | 安全，高效         | 不能跨应用           |
| **EventBus**       | 应用内组件通信   | 解耦，易用         | 需引入第三方库       |
| **RxJava**         | 响应式通信       | 功能强大，链式调用 | 学习曲线陡峭         |
| **LiveData**       | UI与数据通信     | 生命周期感知       | 只能在应用内使用     |
| **接口回调**       | 直接组件通信     | 简单直接，类型安全 | 耦合度高             |

## 十二、常见问题与解决

| 问题             | 现象               | 解决方案                                     |
| ---------------- | ------------------ | -------------------------------------------- |
| **接收不到广播** | 静态注册不生效     | 1. 检查 AndroidManifest 2. 8.0+ 使用动态注册 |
| **内存泄漏**     | 动态注册未取消     | 在对应生命周期取消注册                       |
| **ANR**          | onReceive 执行超时 | 1. 耗时操作放到子线程 2. 使用 goAsync()      |
| **权限问题**     | 安全异常           | 检查权限声明和验证                           |
| **广播冲突**     | 收到意外广播       | 使用更具体的 action                          |

## 十三、性能优化

```
// 1. 使用 goAsync() 处理耗时操作
override fun onReceive(context: Context, intent: Intent) {
    val pendingResult = goAsync()
    
    CoroutineScope(Dispatchers.IO).launch {
        // 执行耗时操作
        processData(intent)
        
        // 完成
        pendingResult.finish()
    }
}

// 2. 批量注册/取消注册
object BroadcastManager {
    private val receivers = mutableMapOf<BroadcastReceiver, IntentFilter>()
    
    fun registerAll(context: Context) {
        receivers.forEach { (receiver, filter) ->
            context.registerReceiver(receiver, filter)
        }
    }
    
    fun unregisterAll(context: Context) {
        receivers.keys.forEach { receiver ->
            context.unregisterReceiver(receiver)
        }
        receivers.clear()
    }
}
```

## 十四、安全注意事项

1. 

   **避免敏感信息**：不在广播中传递密码、token 等

2. 

   **使用权限保护**：对跨应用广播添加权限验证

3. 

   **验证发送方**：检查广播来源

4. 

   **防恶意广播**：设置 exported=false（不需要对外暴露时）

5. 

   **数据验证**：验证接收到的数据格式和内容

## 总结

**广播使用选择决策表**：

| 需求         | 推荐方案                            | 代码提示                              |
| ------------ | ----------------------------------- | ------------------------------------- |
| 监听系统事件 | 动态注册 + 静态注册（仅限豁免列表） | `registerReceiver()`                  |
| 应用内通信   | LocalBroadcastManager               | `LocalBroadcastManager.getInstance()` |
| 跨应用通信   | 带权限的显式广播                    | `sendBroadcast(intent, permission)`   |
| 高频率更新   | EventBus/RxJava/LiveData            | 避免广播性能开销                      |
| 后台接收     | JobScheduler/WorkManager            | Android 8.0+ 替代方案                 |

**核心要点**：

1. 

   尽量使用本地广播进行应用内通信

2. 

   Android 8.0+ 注意隐式广播限制

3. 

   动态注册一定要配对取消注册

4. 

   广播中不要执行耗时操作

5. 

   优先使用有序广播传递处理结果

# 广播接收器（BroadcastReceiver）注册方式对比

下表对比了Android中广播的静态注册（在AndroidManifest.xml中声明）与动态注册（在代码中使用registerReceiver()注册）两种方式的优缺点及适用场景：

| 对比维度         | 静态注册                           | 动态注册                       |
| :--------------- | :--------------------------------- | :----------------------------- |
| **注册位置**     | AndroidManifest.xml                | Java/Kotlin代码                |
| **生效时机**     | 应用安装后即生效                   | 注册代码执行后生效             |
| **生命周期**     | 与应用生命周期一致，从安装到卸载   | 与注册上下文（如Activity）绑定 |
| **系统广播接收** | ✅ 可接收系统广播（即使应用未运行） | ❌ 应用进程必须运行             |
| **开机广播**     | ✅ 可接收`BOOT_COMPLETED`等开机广播 | ❌ 无法接收（需要静态注册）     |
| **资源消耗**     | 较高（常驻内存）                   | 较低（按需注册/注销）          |
| **灵活性**       | 低（固定声明，需重新安装修改）     | 高（可运行时动态控制）         |
| **安全性**       | 较低（其他应用可见）               | 较高（仅本应用内注册）         |
| **代码侵入性**   | 低（声明式配置）                   | 高（需编写注册/注销代码）      |
| **ANR风险**      | 较高（onReceive()需在10秒内返回）  | 可通过`goAsync()`异步处理      |
