# 安卓配置

# LaunchMode

### 四种 LaunchMode 及其作用

#### 1. `standard`（标准模式）

- **定义**：**默认模式**。每次启动该 Activity 都会创建一个**新的实例**，并放入启动它的那个任务栈中。
- **工作原理**：无视是否已存在实例，总是创建新实例。
- **生命周期**：每次都会走完整的 `onCreate()`-> `onStart()`-> `onResume()`。
- **任务栈**：新实例会放入**启动方所在的任务栈**。
- **应用场景**：绝大多数普通 Activity。例如，一个邮件应用的“写邮件”Activity，用户可能同时写多封邮件，每个都是新实例。
- **示例**：一个任务栈的状态可能是 `[MainActivity, DetailActivity, DetailActivity, DetailActivity]`，包含了同一个 Activity 的多个实例。

#### 2. `singleTop`（栈顶复用模式）

- **定义**：如果目标 Activity 的实例**已经位于启动它的任务栈的栈顶**，则不会创建新实例，而是复用这个栈顶实例，并通过 `onNewIntent()`方法将新的 Intent 传递给它。如果不在栈顶，则创建新实例。
- **工作原理**：检查**当前任务栈的栈顶**是否是目标 Activity。
- **生命周期**：
  - 若创建新实例：`onCreate()`-> `onStart()`-> `onResume()`
  - 若复用已有实例：`onNewIntent()`-> `onResume()`
- **任务栈**：新实例（如果需要创建）会放入**启动方所在的任务栈**。
- **应用场景**：防止快速连续点击时多次启动同一个页面。例如，一个通知点击后启动一个 Activity，防止用户连续点击通知打开多个相同的页面。
- **示例**：
  - 任务栈状态为 `[MainActivity, DetailActivity]`（DetailActivity 在栈顶）。
  - 再次启动 `DetailActivity`：会复用栈顶实例，调用其 `onNewIntent()`。栈状态不变。
  - 如果栈状态是 `[DetailActivity, MainActivity]`（DetailActivity 不在栈顶），启动 `DetailActivity`则会创建新实例，栈变为 `[DetailActivity, MainActivity, DetailActivity]`。

#### 3. `singleTask`（栈内复用模式）

- **定义**：系统会为该 Activity 创建一个**独立的任务栈**（如果不存在），并且保证在整个系统中该 Activity **只存在一个实例**。如果已经存在该实例，系统会直接复用该实例（通过 `onNewIntent()`），并**清除该实例之上的所有其他 Activity**，使其回到栈顶。
- **工作原理**：在整个**系统范围内**的**所有任务栈**中寻找目标 Activity 的实例。如果找到，就将其所在的任务栈切换到前台，并清理掉它上面的所有 Activity，然后传递 Intent。如果没找到，就在它指定的任务栈中创建新实例。
- **生命周期**：
  - 若创建新实例：`onCreate()`-> `onStart()`-> `onResume()`
  - 若复用已有实例：`onNewIntent()`-> `onResume()`，并且它上面的 Activity 会被销毁，走 `onDestroy()`。
- **任务栈**：通过 `taskAffinity`属性指定专属任务栈。通常用于定义应用的**主入口**（如主界面）。
- **应用场景**：应用的主页（Home Page）。例如，一个应用深处多个页面，点击通知后希望直接返回主页并清除上面的所有页面，而不是在返回时一层层后退。
- **示例**：
  - 用户操作路径：`主页(SingleTask) -> 列表页 -> 详情页 -> 分享页`。
  - 此时在分享页，又收到了一个通知，点击通知再次启动 `主页`。
  - **结果**：系统会找到已有的主页实例，并清除它上面的所有页面（列表页、详情页、分享页都被销毁），主页显示到前台，并接收新的 Intent。最终栈状态为 `[主页]`。

#### 4. `singleInstance`（单实例模式）

- **定义**：最极端的模式。系统会为该 Activity 创建一个**全新的、独立的任务栈**，并且该任务栈中**有且仅有这一个 Activity 实例**。如果该实例已存在，则直接复用该实例（通过 `onNewIntent()`）。
- **工作原理**：保证一个 Activity 独占一个任务栈，并且该栈中只有它自己。
- **生命周期**：同 `singleTask`。
- **任务栈**：**强制**运行在一个独立的任务栈中。
- **应用场景**：需要与其他应用共享、完全隔离的 Activity。例如，来电界面、闹钟提醒界面、或者一个专门用于分享的 Activity。这样它就可以单独出现在最近任务列表中。
- **示例**：应用 A 启动了一个 `SingleInstance`的 Activity `X`。此时最近任务列表里会看到两个任务：一个是应用 A 的任务栈，另一个是单独的任务栈，里面只有 `X`。

#### 使用方式

```xml
<activity
    android:name=".DetailActivity"
    android:launchMode="singleTop" />
```
